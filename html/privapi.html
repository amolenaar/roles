Private API¶

For those interested in (non-public) API.

Utility classes¶

class roles.role.cached¶

Cache the output of the function invocation.

>>> @cached
... def cap(s): return s.upper()
>>> cap('a')
'A'
>>> cap('b')
'B'

Show cache contents:

>>> cap.cache
{('a',): 'A', ('b',): 'B'}

Clear the cache:

>>> cap.cache.clear()
>>> cap.cache
{}

Due to the caching, we can not take key-value arguments:

>>> cap(s='a')     
Traceback (most recent call last):
  ...
TypeError: wrapper() got an unexpected keyword argument 's'
roles.role.class_fields(*args)¶

Get all fields declared in a class, including superclasses.

Don’t forget to clear the cache if fields are added to a class or role!

Generic role factory¶

class roles.factory.RoleFactoryType¶

RoleFactoryType is a special kind of RoleType: with the @assignto class decorator this class is applied to any RoleType instance.

Now subroles are able to be automatically applied to specific instances. Thus, the Role class is acting as a factory for it’s own types.

Note that this metaclass is automatically applied the first time @assignto is used to decorate a role class. There is no need to assign RoleFactoryType explicitly.

lookup(*args)¶

Find a specific Role type for a subject. The returned role class is a subclass of the factory role.

register(cls, rolecls, strict)¶

Register a new roleclass for a specific class.

Note: strict is set if the topmost role has assignto() applied.
